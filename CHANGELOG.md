## [v7.0.0]

### ✨ 新增
- **重启 Hermes 指令**：新增 `/重启爱马仕`（别名：`/重启Hermes`），可在 QQ 群聊或私聊中一键重启 Hermes Agent
  - 指令执行后立即反馈 "🔄 正在重启Hermes…" 提示
  - 重启命令在子线程中异步执行，不阻塞 AstrBot 主线程
  - 内置 30 秒超时保护，避免命令卡死
  - 重启失败时自动反馈错误信息
  - 重启命令可在配置 `重启Hermes命令` 中自定义（默认 `systemctl restart hermes`）
  - 重启成功后自动发送连接报告

- **Hermes 连接报告**：WebSocket 成功连接到 Hermes 时，自动发送连接状态报告
  - 常规连接报告：显示连接时间和状态（"✅ 已连接到Hermes"）
  - 重启后连接报告：显示重启耗时（"✅ Hermes重启完成" + 耗时秒数）
  - 支持发送到群聊或私聊（配置 `发送目标类型` + `发送目标ID`）
  - 若通过重启指令触发，报告自动回复到发起指令的聊天室
  - 配置项 `启用Hermes连接报告` 控制是否开启此功能（默认关闭）

### 📦 新增配置项（其他配置）
| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `启用Hermes连接报告` | bool | `false` | 是否在 WebSocket 连接成功时发送报告 |
| `发送目标类型` | string | `群聊` | 连接报告的发送目标类型（群聊/私聊） |
| `发送目标ID` | string | `""` | 连接报告的发送目标 QQ 号或群号 |
| `重启Hermes命令` | string | `systemctl restart hermes` | `/重启爱马仕` 指令执行的 shell 命令 |

## [v6.8.1]

### ✨ 新增
- 触发关键词支持参数化匹配，每个关键词可附加以下参数：
  - `-s`：以前缀方式匹配（消息以该关键词开头才触发）
  - `-e`：以后缀方式匹配（消息以该关键词结尾才触发）
  - `-d`：自动去除关键词（触发后从消息中移除该关键词再转发）
  - 参数可组合使用，如 `hello -s -d` 表示前缀匹配且自动去除，`bye -e -d` 表示后缀匹配且自动去除
  - ⚠️ 注意：参数必须用空格分隔，不能写成 `-sd`，必须写 `-s -d`
  - ⚠️ 注意：`-s` 和 `-e` 不能同时使用（一个关键词不能同时是前缀又是后缀）
- 新增 `去除关键词` 方法，支持前缀/后缀/包含三种去除模式，同时处理 message 链和 raw_message

## [v3.3] - 2026-04-24

### 🏗️ 模块化重构

将单体文件拆分为独立模块：

- `command_cache.py` - 指令缓存、查找、别名解析、黑白名单

- 指令缓存重建时机优化
- 日志输出调整
