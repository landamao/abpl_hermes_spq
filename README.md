## 🔮 核心愿景：AI 双模型协同的终极形态

传统 QQ 机器人 + AI 方案只有“聊天”这一种形态：

- **AstrBot**：拥有丰富的即时交互能力——点歌、表情包、宠物养成、群管理……但无法执行复杂多步骤任务。
- **Hermes Agent**：具备代码执行、网页浏览、持久记忆、定时任务等深度能力，但缺少与 QQ 生态的亲密互动。

**这个适配器彻底打破边界**：让两者不再是孤岛，而是一个**协同工作的双核 AI 系统**——AstrBot 处理即时轻量任务，Hermes 处理深度复杂任务，并且**互相调用对方的能力**。  
更引入了 **企业级敏感信息自动过滤**、**免打扰审批自动艾特**、**反向HTTP全功能代理** 等工业级特性，让机器人运营者彻底解放双手。

### 四大核心引擎

#### ① 智能消息路由与分流
- 支持**关键词触发**、**@触发**、**引用唤醒**、**指令前缀触发**、**全量转发**（私聊）等多种模式
- **冲突处理策略**：当 LLM 和 Hermes 都想回答时，可选择：
  - `只用Hermes`：拦截 LLM，仅 Hermes 回答（避免重复）
  - `不使用Hermes`：仅本地 LLM 处理
  - `都处理`：双模型同时响应
- **双重白名单**：群白名单 + 用户白名单，私聊独立控制，安全灵活

#### ② 指令代理引擎（v6.0 重大更新）
Hermes 可以像用户一样调用 AstrBot **所有插件的指令**——不是模拟，而是**直接调用 AstrBot 内部的指令处理器**，自动匹配群聊/私聊上下文，执行结果实时返回。

```bash
# Hermes 发起一个 HTTP 请求，AstrBot 就像收到了一条真实指令
curl -X POST http://127.0.0.1:6700/api/execute \
  -H "Authorization: Bearer your_token" \
  -H "Content-Type: application/json" \
  -d '{
    "command": "点歌",
    "args": "让风告诉你",
    "group_id": "947928324"
  }'
```

支持指令扫描、缓存、别名解析、黑白名单控制，并提供完整的 REST API 供 Hermes 查询指令列表、执行指令。

#### ③ LLM 工具集（AstrBot AI 可主动调用）
当 AstrBot 自身接入了 LLM（如 DeepSeek、GPT），AI 可以主动调用四个工具：

| 工具 | 能力 |
|------|------|
| `hermes_agent` | 让 Hermes 执行任意任务或指令 |
| `execute_command` | 执行任意 AstrBot 插件指令（由 LLM 自主决定） |
| `hermes_status` | 查询 Agent 运行状态（连接、缓存等） |
| `list_commands` | 获取所有可用指令菜单 |

→ AstrBot 的 AI 可以自己判断“这件事我搞不定，让 Hermes 来”，然后自动调用工具，实现真正的智能分工。

#### ④ 🔐 安全过滤引擎（自动脱敏，防泄漏）
**这是企业级机器人的必备能力**：所有流出到 QQ 的消息，都会经过敏感字符过滤。

- **自动扫描密钥**：读取 `（框架目录）/data/cmd_config.json` 中的所有 API Key，以及 Hermes `config.yaml` 中的所有 `api_key` 字段，自动加入敏感字符列表。
- **自动添加 Token**：连接配置中的 `hermes_token`、`NapCat_api_token`、`反向HTTP令牌`、`http指令服务器token` 全部自动纳入过滤。
- **实时替换**：消息中的敏感字符被替换为 `***`（可自定义），彻底防止 API Key 泄漏到群聊。
- **过滤框架 LLM 输出**：不仅过滤 Hermes 的回复，**AstrBot 自身 LLM 的生成结果也会被过滤**——双重保险，绝无遗漏。

> 💡 你甚至不需要手动配置任何敏感词——适配器会自动扫描你的所有配置文件，把密钥全部保护起来。

## 🛡️ 安全与免打扰体系

### 免打扰审批 + 自动艾特
作为群主/管理员，你希望 Agent 自主工作，又不想被频繁打扰，但危险命令又必须有人审批。适配器提供**完美方案**：

- **Agent 静默工作**：普通对话无需人工介入，Agent 自动完成。
- **危险命令自动拦截**：未授权的 `/approve` 或敏感指令，会被自动拒绝（可配置为拒绝并发送自定义消息）。
- **自动艾特管理员**：当消息包含 `"Dangerous command requires approval"` 等关键词时，**自动在群内艾特指定管理员**，提醒审批。
- **关键词自定义**：你可以在配置中修改触发艾特的关键词列表，适应不同场景。

> **效果**：你只需要躺在椅子上喝茶，Agent 自己干活；真有危险操作，QQ 会弹窗艾特你，一键审批。

### 🔄 反向 HTTP 服务 —— Agent 自由调用 NapCat API，且保留敏感过滤

Hermes 不仅可以通过 WebSocket 接收消息，还能**主动调用 NapCat 的所有 API**（获取群列表、发送私聊、禁言、撤回等）。  
- 适配器启动一个**独立的反向 HTTP 服务器**（默认端口 `5701`），完全兼容 OneBot v11 规范。  
- **关键优势**：所有通过此入口的请求（包括发送的消息）**仍会经过敏感字符过滤**，确保任何出口信息都不会泄露密钥。  
- 推荐与“框架已有的WebSocket”发送方式配合使用——既能享受框架原生 WebSocket 的稳定，又能利用反向 HTTP 作为 NapCat 的代理，实现**完整的敏感词过滤闭环**。

> 🔥 **推荐**：反向 HTTP + 框架 WebSocket 的组合，既稳定又安全，是生产环境的最佳实践。

### 指令前缀自定义 + 中文映射
- **避免指令冲突**：默认指令前缀 `/` 可与 AstrBot 的唤醒前缀完全独立，你甚至可以设为空（使用默认 `/`）。
- **中文映射**：管理员可以用 `/同意`、`/一直同意`、`/会话同意` 等自然语言代替 `/approve`，用户无需记忆英文命令。映射规则完全可配置。

### 场景三：免打扰审批 + 自动艾特
```
普通群友：@机器人 给我执行 /restart_server

```
🤫 **管理员不需要盯着屏幕，Agent 也不需要停下工作**——完美实现免打扰运维。

```
→ 群内看到的是：“错误：你的 openai_api_key=*** 无效”
```
🔒 **密钥永远不会泄漏到群聊**，连框架自身的 LLM 回复也被过滤。

### 场景五：反向 HTTP —— 无缝接入
```

→ 适配器反向 HTTP 接收请求
→ 自动执行敏感词过滤、自动艾特、指令代理
→ 通过框架 WebSocket 发送到 QQ
```
🌉 **Hermes 无需任何改造，直接享受全套增强功能**。

## 📡 完整 API 矩阵

适配器暴露了完整的 RESTful API（默认端口 `6700`），让 Hermes 可以：

| 端点 | 方法 | 能力 |
|------|------|------|
| `GET /api/health` | GET | 健康检查 + 连接状态 + 指令缓存数量 |
| `GET /api/commands` | GET | 获取所有指令列表（按插件分组） |
| `GET /api/commands/for_hermes` | GET | 为 Agent 优化的扁平指令清单，含描述和插件名 |
| `GET /api/command/{name}` | GET | 查询单个指令详情（别名、权限、示例） |
| `POST /api/execute` | POST | 执行任意 AstrBot 指令（需 Bearer Token） |

**执行请求示例**：
```json
POST /api/execute
{
  "command": "点歌",
  "args": "晴天",
  "group_id": "123456789",
  "user_id": "987654321",
  "user_name": "用户昵称"
}
```

响应中包含执行结果（文本、图片、发送状态）。

| | `自动添加密钥` | **自动扫描 AstrBot 和 Hermes 配置中的 API Key，加入过滤** |
| | `自动添加token` | 自动添加连接配置中的各类 Token |
| | `过滤llm结果` | **同时过滤框架 LLM 生成的文本** |
| | `替换目标` | 敏感词替换为 `***` 或留空（直接删除） |
| **指令配置** | `启用指令HTTP服务器` | 是否开启 `/api/execute` 端点 |
| | `指令白名单` / `指令黑名单` | 限制 Hermes 可调用的指令 |

- 想严格管控？设置群白名单 + 用户白名单 + 指令黑名单 + API Token + 敏感词自动过滤 + 自动艾特审批
- 想开放体验？全部留空，配合冲突策略调优
- 想全量智能？私聊开启 `私聊转发所有消息`，让 Agent 感知所有对话
- 想免打扰运维？开启反向 HTTP，设置自动艾特管理员，Agent 静默工作

## 📋 详细操作指引（从零开始）

### 前置要求
- 已部署 AstrBot，且使用 NapCat（或兼容 OneBot v11 的实现）连接 QQ
- 已安装本插件到AstrBot

### 第一步：安装 Hermes Agent 本体
如果你还没有安装 Hermes，请先访问 **[Hermes Agent 官方仓库](https://github.com/NousResearch/hermes-agent)**，按照其中的说明进行安装并配置AI。

### 第二步：安装 Hermes QQ OneBot 适配器
Hermes 本身不直接支持 QQ 协议，需要通过 **[Hermes QQ OneBot 平台适配器](https://github.com/chrysoljq/hermes_qq_onebot)** 进行桥接。

1. 克隆或下载该适配器仓库：
   ```bash
   git clone https://github.com/chrysoljq/hermes_qq_onebot.git
   cd hermes_qq_onebot
   ```
2. 按照其 README 中的依赖安装说明，配置必要的 Python 环境。
3. 阅读该仓库的配置文档，正确填写配置文件（通常是一个 `config.yaml` 或 `config.json`）。你需要重点关注以下内容：
   - **OneBot 反向 WebSocket 监听端口** （本插件作为 **OneBot 反向 WebSocket适配器**）
   - **模型/LLM 配置**：将 Hermes 使用的底层大模型配置正确（API key、模型名称等）

### 第三步：启动 Hermes 网关
```bash
hermes gateway restart   # 或根据其文档使用特定命令
```

### 第四步：配置本插件的连接信息
打开 AstrBot WebUI，找到 “Hermes适配器” 插件配置页面

- **`Hermes WebSocket 地址`**：填写 `ws://127.0.0.1:6701`（与第二步中 Hermes OneBot 适配器的 WebSocket 端口一致）
- **`消息发送方式`**：选择 `框架已有的WebSocket`（推荐，配合反向HTTP可实现全部增强功能）
- **`开启反向HTTP`**：勾选 **是**（强烈推荐）
- **`反向HTTP端口`**：默认 `5701`，保持与 Hermes OneBot 适配器配置的 HTTP 回调地址一致
- **`HTTP 指令服务器端口`**：默认 `6700`，供Hermes调用指令API
- **`自动艾特用户`**：填写你的 QQ 号，审批时自动艾特你
- **`触发艾特关键词`**：可自定义，默认包含 `"Dangerous command requires approval"`
- 根据需求设置消息过滤规则（关键词、白名单等）

### 第五步：启动并验证连接
1. 重启 AstrBot 或重载插件。
2. 在 AstrBot 日志中查找以下关键输出：
   - `"正在连接到 Hermes: ws://127.0.0.1:6701"`
   - `"WebSocket 已连接到 Hermes"`
   - `"反向HTTP服务器已启动: http://0.0.0.0:5701"`（如开启）
3. 在 QQ 群中发送配置好的触发关键词，例如 “纳西妲 你好”，观察 Hermes 是否在思考，是否返回回复。

至此，双核协同系统部署完成。你可以测试：
- 简单指令（点歌、宠物）由 AstrBot 直接处理
- 复杂任务（写脚本、爬取信息）由 Hermes Agent 自主执行
- 发送 `/hermes status` 查看状态
- 尝试一个需要审批的命令，验证自动艾特

> 💡 提示：如果连接失败，请检查防火墙、端口是否被占用，以及 Hermes OneBot 适配器的 WebSocket 地址与端口是否与本插件配置完全一致。

## 🔒 安全第一

内置多层防护：

1. **API 层**：Bearer Token 认证（`http指令服务器token`）
2. **传输层**：WebSocket Access Token（`hermes_token`）
3. **指令层**：黑/白名单（防止执行 `重启`、`关机` 等敏感指令）
4. **用户层**：群白名单 + 用户白名单（`允许的群组` / `允许的用户`）
5. **审批层**：`/approve` `/deny` 权限用户限定，支持自动艾特提醒
6. **脱敏层**：自动过滤所有回复中的敏感信息（API Key、Token 等），**包括框架 LLM 输出**
7. **免打扰层**：危险命令自动拒绝，管理员无需实时监控

## 💡 为什么这是革命性的？

传统的 QQ 机器人 + AI 方案是**单线程**的：

```
用户 → 机器人 → LLM → 回复
```

而本适配器实现的是**双核并行 + 能力互通 + 企业级安全**：

```
用户 → AstrBot（即时响应）
         ↓ 遇到复杂任务
       Hermes Agent（深度执行）
         ↓ 需要群聊能力
       AstrBot 插件生态（点歌/画图/管理）
         ↓
      安全过滤引擎（自动脱敏）→ 免打扰审批（自动艾特）→ 统一回复
```

**快任务走快通道，慢任务走深通道，能力互通，用户无感。**  
**敏感信息自动隐藏，危险命令自动拦截 + 艾特管理员，Agent 静默工作。**  
**AstrBot 自身的 LLM 也可以主动调用 Hermes 和所有插件指令**——AI 不再是被动应答，而是能自主编排任务、调用工具的智能中枢。
